# MedInsight AI

MedInsight AI is a student-friendly medical RAG chatbot built with Streamlit,
FAISS, LangChain, ONNX embeddings, and Groq. It retrieves relevant medical
content from a local knowledge base and uses an LLM to produce a structured
answer with source snippets.

This project is for learning and portfolio use only. It is not a medical device
and must not be used for diagnosis, treatment decisions, or emergency advice.

## Features

- Streamlit chat interface
- Local FAISS vector search
- ONNX embeddings using `all-MiniLM-L6-v2`
- Groq-hosted LLM responses
- Source snippets with relevance scores
- Session chat history and chat export
- One-time knowledge base builder for PDF, XML, and CSV data

## Architecture

```text
User question
    -> ONNX embedding model
    -> FAISS vector store
    -> top medical context chunks
    -> Groq LLM
    -> Streamlit answer with sources
```

## Project Structure

```text
MedInsightAI/
|-- .streamlit/
|   `-- config.toml
|-- data/
|   |-- *.pdf
|   |-- *.csv
|   `-- medical QA XML folders
|-- vectorstore/
|   `-- db_faiss/              # generated by create_memory_for_llm.py
|-- embeddings.py              # shared ONNX embedding helper
|-- medibot.py                 # main Streamlit app
|-- create_memory_for_llm.py   # builds the FAISS knowledge base
|-- connect_memory_with_llm.py # optional CLI tester
|-- download_model.py          # downloads ONNX embedding files
|-- requirements.txt
|-- pyproject.toml
|-- Pipfile
|-- .env.example
`-- README.md
```

Generated folders:

- `.model_cache/` stores downloaded embedding model files.
- `vectorstore/db_faiss/` stores the generated FAISS index.
- `logs/` stores local runtime logs.

## Local Setup

### 1. Create and activate a virtual environment

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

macOS/Linux:

```bash
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Add your API key

```bash
copy .env.example .env
```

On macOS/Linux:

```bash
cp .env.example .env
```

Edit `.env`:

```env
GROQ_API_KEY=your_groq_api_key_here
HF_TOKEN=optional_huggingface_token_here
```

`HF_TOKEN` is optional for public model download, but it can help avoid rate
limits.

### 4. Download the embedding model

```bash
python download_model.py
```

### 5. Build the knowledge base

Make sure the PDFs, XML folders, and CSV files are inside `data/`, then run:

```bash
python create_memory_for_llm.py
```

This creates:

```text
vectorstore/db_faiss/index.faiss
vectorstore/db_faiss/index.pkl
```

### 6. Run the app

```bash
streamlit run medibot.py
```

Open the local URL shown in the terminal, usually:

```text
http://localhost:8501
```

## Streamlit Community Cloud Deployment

For Streamlit Cloud, the app needs these project files in GitHub:

- `medibot.py`
- `embeddings.py`
- `requirements.txt`
- `.streamlit/config.toml`
- `.model_cache/` with the ONNX model files
- `vectorstore/db_faiss/index.faiss`
- `vectorstore/db_faiss/index.pkl`

Deployment steps:

1. Run `python download_model.py` locally.
2. Run `python create_memory_for_llm.py` locally.
3. Commit the generated `.model_cache/` and `vectorstore/db_faiss/` files.
4. Push the project to GitHub.
5. In Streamlit Cloud, set the main file path to `medibot.py`.
6. Add this secret in Streamlit Cloud settings:

```toml
GROQ_API_KEY = "your_groq_api_key_here"
```

If the model or vector store is missing, the app shows a clear setup message
instead of crashing.

## Dependency Notes

The app intentionally avoids these heavy runtime dependencies:

- `torch`
- `sentence-transformers`
- `langchain-huggingface`

Embeddings are handled with `onnxruntime` and `tokenizers`, which keeps the app
lighter for Streamlit Cloud and normal student laptops.

## GitHub Notes

- Do not push `.env`, `.venv`, `logs`, cache files, or editor settings.
- Raw data files are ignored by default because the folder is large.
- The generated FAISS files are needed for Streamlit Cloud unless you rebuild
  the index another way during deployment.
- `index.faiss` is close to GitHub's 100 MB file limit. Use Git LFS if GitHub
  blocks the upload.
